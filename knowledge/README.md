# Knowledge Folder

This folder contains project-specific empirical research context. AI agents
should read these files before planning or implementing non-trivial tasks so
workflow rules stay separate from research substance.

Read in order:

1. README.md
2. project.md
3. findings.md
4. ideas.md

## What Belongs Here

- `project.md`: research question, motivation, literature, data, variables,
  treatment, outcomes, identification strategy, models, hypotheses, open
  questions, things to avoid, and timeline.
- `findings.md`: what the project has learned so far (established empirical results), with evidence, source
  outputs, robustness status, and caveats.
- `ideas.md`: evolving hypotheses, questions, and rough thoughts

## What Does Not Belong Here

- General AI-agent workflow rules belong in `AGENTS.md`.
- Coding style, path, language, and Makefile conventions belong in
  `code/AGENTS.md`.
- Durable workflow lessons belong in `MEMORY.md` using structured `[LEARN]`
  entries.
